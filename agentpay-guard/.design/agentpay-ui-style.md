# AgentPay Guard UI Style

## Product Personality

AgentPay Guard is an institutional fintech/regtech evidence-room interface. It should feel calm, precise, operational, and high-trust.

## Avoid

- Generic AI dashboard visuals.
- Crypto-bro motifs, neon color, cartoon robots, coin art, fake vanity metrics.
- Oversized marketing hero sections inside the app.
- Heavy gradients, decorative blobs, or vague feature cards.
- Claims that imply full KYA, KYB, AML, sanctions, legal, or regulatory approval.

## Layout Principles

The primary screen should tell an evidence story:

Principal -> Approver -> Agent -> Mandate -> Payment Request -> Decision -> Receipt or Case -> Audit Evidence.

Prefer dense but readable information. Keep the most important state visible without requiring scrolling: decision, reason, amount, merchant, authority chain, rule checklist, audit verification, and caveat.

## Visual Language

- Off-white workspace background.
- White panels with fine borders and restrained shadow.
- Dark left navigation for product context.
- Green only for approved.
- Red only for blocked.
- Amber only for pending approval or manual review.
- Blue/gray for evidence, audit, and neutral controls.
- Monospace for hashes, IDs, timestamps, and policy versions.
- Tabular numbers for amounts and timestamps.

## Components

- Sidebar navigation.
- Case summary panel.
- Compact action buttons.
- Authority chain.
- Rule checklist.
- Audit timeline.
- Evidence tabs.
- Caveat/status panel.

## Copy Style

Use plain compliance-friendly language. Be exact about what the app proves and what it does not prove. Always preserve the mock-payment caveat.
